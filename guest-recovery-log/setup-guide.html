H.E.A.R.D. Log — Setup Guide

From zero to a working guest-recovery log. No coding experience needed — you'll copy and paste a few things and click some buttons. Follow the steps in order.

⏱ About 15 minutes
You'll set up three things:
  1. A Google Sheet to hold the data
  2. The Apps Script backend that reads and writes to that sheet
  3. The frontend (index.html) your managers actually use

Before you start, download Code.gs and index.html from this project, and have a Google account that can create Google Sheets (your CFA Workspace account is perfect).

1

Create the Google Sheet

  1. Go to sheets.google.com and click Blank spreadsheet.
  2. Rename it something clear, like "H.E.A.R.D. Log Data" (click the title at top-left).
  3. Leave the default tab named Sheet1. If yours is named something else, that's fine — you'll match it in Step 3.

You don't need to add column headers — the setup function does that for you in Step 3.

2

Get the Sheet ID

The Sheet ID is the long code in the sheet's web address:

https://docs.google.com/spreadsheets/d/1AbC...XyZ/edit#gid=0
                                        └──── this part ────┘
  1. Copy everything between /d/ and the next /.
  2. Keep it handy — you'll paste it in the next step.
3

Set Up the Backend & Run One-Click Setup

  1. In your Sheet, click Extensions → Apps Script. A code editor opens.
  2. Delete any starter code shown there (usually an empty myFunction).
  3. Open Code.gs from this project, select all, copy, and paste it into the editor.
  4. Near the top, find const SHEET_ID = 'YOUR_SHEET_ID'; and replace YOUR_SHEET_ID with the ID from Step 2 (keep the quotes).
  5. If your tab isn't named Sheet1, also update const SHEET_NAME = 'Sheet1';.
  6. Click the 💾 Save icon.

Now run the one-click setup:

  1. In the function dropdown at the top (may say doGet), choose runFirstTimeSetup.
  2. Click ▶ Run.
  3. The first time, Google asks for permission: Review permissions → pick your account → Advanced → Go to (project) → Allow.
  4. Open the Execution log at the bottom. You should see "H.E.A.R.D. Log setup complete."
Safe to re-run. runFirstTimeSetup never overwrites or deletes existing data — it just makes sure the data tab, header row, and alert storage all exist.
4

Deploy the Backend as a Web App

  1. In the Apps Script editor, click Deploy → New deployment (top-right).
  2. Click the gear ⚙️ next to "Select type" and choose Web app.
  3. Set Execute as: Me and Who has access: Anyone — this matters, or the app can't reach it.
  4. Click Deploy and copy the Web app URL (ends in /exec). Keep it for the next step.
Updating later: if you change Code.gs, go to Deploy → Manage deployments → ✎ Edit → Version: New version → Deploy. The URL stays the same.
5

Configure the Frontend

  1. Open index.html in a plain text editor (TextEdit, Notepad, or VS Code).
  2. Find const APPS_SCRIPT_URL = 'YOUR_APPS_SCRIPT_URL'; and paste your Web app URL from Step 4 (keep the quotes).
  3. Find if (passcode === 'YOUR_LOGIN_PASSCODE') and set the 4-digit code your team uses to log in (e.g. '1234').
  4. Find if (settingsPasscode === 'YOUR_SETTINGS_PASSCODE') and set a different 4-digit code only managers know — it protects the email-alert settings (e.g. '5678').
  5. Save the file.
6

Use It

  1. Double-click index.html to open it in any browser (or host it — see below).
  2. Enter your login passcode from Step 5.
  3. Log a test complaint: type a phone number, fill the required fields (marked with *), click Submit Complaint.
  4. Open your Google Sheet — the row should appear. 🎉

Email Alerts for Repeat Complainers

  1. In the app, open ⚙️ Settings and enter your settings passcode (Step 5).
  2. Add the email addresses that should be notified when a guest crosses the repeat-complaint threshold.
  3. Addresses are stored on the backend (Script Properties), so they persist for everyone — not just your device.

Troubleshooting

"Set SHEET_ID at the top of this file before running setup." — You ran runFirstTimeSetup before pasting your Sheet ID (Step 3). Paste it, save, run again.

App opens but data won't load / submit fails — Check APPS_SCRIPT_URL exactly matches the URL from Step 4 (must end in /exec, not /dev). Confirm the deployment's Who has access is Anyone. After any Code.gs change, redeploy a New version.

"Google hasn't verified this app" — Normal for personal scripts. Click Advanced → Go to (project) → Allow. It's your own script.

Rows appear but columns look shifted — Re-running runFirstTimeSetup on an empty sheet rewrites the headers correctly. It won't touch a sheet that already has data — clear the tab first for a clean reset.

Wrong tab — SHEET_NAME in Code.gs must exactly match your tab name, including capitalization.
