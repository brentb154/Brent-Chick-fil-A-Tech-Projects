H.E.A.R.D. Log

Guest-recovery complaint log. Log guest issues in under a minute, auto-flag repeat complainers via phone lookup, see a live dashboard of trends and resolutions, and send email alerts — all backed by a single Google Sheet.

React Google Apps Script Sheets Integration
This is a setup-it-yourself template. The frontend (index.html) is wired to a Google Sheet through a Google Apps Script web app. Before it will load or save anything, you deploy your own copy — your Sheet, your script, your passcodes. Nothing here connects to a live store. The full walkthrough takes about 15 minutes.
📖 Open Setup Guide 📖 Go deeper: what it is & how it works →

What It Does

  • One-tap complaint form — phone, guest, order details, issue type, resolution, daypart, manager
  • Live phone lookup flags repeat complainers as you type (last 60 days)
  • Dashboard with totals, flagged-guest count, and trends
  • Searchable, filterable log — edit, delete, and mark-resolved on any entry
  • Configurable email alerts for repeat complainers (passcode-protected settings)
  • Offline mode — complaints save locally and submit automatically when back online
  • One-click runFirstTimeSetup() builds the sheet, headers, and alert storage for you

Repeat-Complainer Flags

⚠ Yellow Badge

Guest has 2 complaints in the last 30 days — the one you're logging makes 3.

⚠ Red Badge

Guest has 3+ complaints in the last 30 days. Tap the badge to see their history.

How It Works

  • Frontend — index.html, a self-contained React app (no build step). Open in any browser or pin to a tablet.
  • Backend — Code.gs, a Google Apps Script web app bound to your Sheet. Handles all reads and writes.
  • Data — one Google Sheet tab is the database. Columns are built automatically on first setup.
