Payroll System — Setup Guide

Written for zero technical experience. This is the biggest tool in the set, so take it slow — follow each step exactly and you'll have a working payroll companion in about 30–45 minutes.

⏱ About 30–45 minutes
This system lives inside one Google Sheet and opens as a web app. It tracks overtime, manages uniform orders and payroll deductions, and handles PTO requests — with employee self-service forms you can put behind a QR code. It sits next to your real payroll provider and gives you the visibility they don't; it doesn't replace them.
Use a computer, not a phone or tablet, for setup. Chrome works best. Have the project files open in a folder on your computer before you start.
1

Create the Spreadsheet

  1. Go to drive.google.com and sign in.
  2. Click + New → Google Sheets.
  3. Name it something like "Payroll System 2026".
This one spreadsheet stores all your data — orders, employees, OT records. Don't delete it, and don't rename the tabs the setup creates.
2

Open the Script Editor

  1. In your spreadsheet, click Extensions → Apps Script.
  2. A new tab opens with a code editor and a default Code.gs file.
3

Create All the Files

This is the longest part — there are about 26 files. It's repetitive, not hard. For each one: + next to "Files" → pick Script or HTML → name it exactly (no extension) → paste the matching file's contents → Ctrl/Cmd+S to save.

Script files (4):

Code (already there)   DashboardModule   PTOModule   ReportsModule

Core HTML files (6):

Index   MainApp   MainAppJS   Styles   JavaScript   View_Help

View files (13):

View_Dashboard          View_OTUpload         View_OTHistory
View_OTEmployee         View_OTTrends         View_UniformOrders
View_UniformDeductions  View_UniformCatalog   View_PTORecords
View_PayrollProcessing  View_SettingsPage     View_SystemHealth
View_PayrollCalendar    View_YearEndWizard

Employee forms (2, bilingual):

EmployeeUniformRequest   EmployeePTORequest
Two things that bite people here: some files are very long — copy all the content. And file names must match exactly, capitalization included, or the page comes up blank.
4

Initialize the System

  1. Click into Code.gs. In the function dropdown at the top, choose manualInit.
  2. Click ▶ Run.
  3. First run asks for permission: Review permissions → your account → Advanced → Go to Payroll System → Allow.

The Execution log should list the sheets it built (OT_History, Employees, Settings, and more). That's setup done.

Safe to re-run. manualInit creates anything that's missing without wiping existing data.
5

Deploy the App

  1. Click Deploy → New deployment (top-right).
  2. Gear ⚙ next to "Select type" → Web app.
  3. Set Description: Version 1.0, Execute as: Me, Who has access: Anyone (or Anyone within your domain).
  4. Click Deploy, authorize again if asked, and copy the Web app URL (ends in /exec). Bookmark it — that's your app.
  5. Open the URL in a new tab. First load is slow (10–20 sec); then you'll see the Dashboard.
Updating later: after any code change, Deploy → Manage deployments → ✎ Edit → Version: New version → Deploy. The URL stays the same. Saving code alone changes nothing until you redeploy.
6

Configure Settings & Passcode

In the app, open Settings (left sidebar):

Toggle email notifications on, then click Save All Settings.

Manager passcode (gates the receiving panel): go back to the Google Spreadsheet (not the app), open the PAYROLL_SETTINGS tab, find ManagerPasscode, and enter a number like 1234.

7

Add Employees

Easiest: go to Overtime → Upload Data and upload your CSV from CFA Home — employees are created automatically.

Or manually: in the Sheet's Employees tab, add rows with Employee_ID, Full_Name, Primary_Location, Status (Active).

8

Set Up the Employee Forms (QR Codes)

Employees order uniforms and request PTO through two self-service pages. Take your Web app URL and add the page on the end:

Uniform request:  YOUR-APP-URL?view=uniform-request
PTO request:      YOUR-APP-URL?view=pto-request
Use ?view=, not ?page=. The app routes these forms off the view parameter — an older note used ?page=, which does nothing. If a form link opens the dashboard instead of the form, this is why.

Make QR codes for each at qr-code-generator.com and post them in the break room / by the time clock.

Daily Workflow

  • Monday — upload OT data from CFA Home.
  • As needed — process uniform orders, approve PTO.
  • Before payroll — run Pre-Payroll Validation, process deductions.

Troubleshooting

"Script function not found: doGet" — Code.gs didn't copy fully. Re-copy the entire file, save, and redeploy a new version.

White / blank screen — the Index HTML file is missing or incomplete, or a file name's capitalization is off. Confirm Index exists with content, then redeploy.

"Authorization required" keeps appearing — run any function once from the editor and complete the Allow flow again.

Changes aren't showing up — you saved but didn't redeploy. Deploy → Manage deployments → Edit → New version → Deploy.

A form link opens the dashboard, not the form — check the URL uses ?view=uniform-request / ?view=pto-request (Step 8).

Emails aren't sending — check the addresses, check spam, and remember Google caps at roughly 100 emails/day.

Not sure what's wrong — open System Health in the app; it runs built-in diagnostics.
