Payroll Management System

Complete payroll solution with PTO tracking, overtime management, uniform ordering, and year-end processing. Full dashboard web app with multiple views.

Google Apps Script Google Sheets Web App
⚠️ Real talk before you start this one. Of everything I've built, this is the most involved tool — it's about 26 files, a real web-app deploy, and it handles your team's actual time-punch and payroll data. I run it at both my restaurants and it works great, and the guide below walks through every step. But if you're not fully comfortable setting up a Google Apps Script web app, there's no shame in taking your time, grabbing someone who is, or sitting this one out. Nothing here will hurt your data — just be confident in yourself before you lean on it for live payroll.
This is a Google Apps Script application. It runs inside Google Sheets and cannot be opened directly in a browser. This is the biggest tool in the set, so the full step-by-step guide takes it slow — about 30–45 minutes, written for zero technical experience.

Full Step-by-Step Setup Guide

Written for zero technical experience. This is the biggest tool in the set, so take it slow — about 30–45 minutes. It sits next to your real payroll provider and gives you visibility they don't; it doesn't replace them. Use a computer (not a phone).

Two things that bite people: some files are very long — copy all the content; and file names must match exactly, capitalization included, or the page comes up blank.

1. Create the Spreadsheet

At drive.google.com, click + New → Google Sheets and name it "Payroll System 2026". This one Sheet stores all your data — don't delete it or rename the tabs the setup creates.

2. Open the Script Editor

In the Sheet, click Extensions → Apps Script. A code editor opens with a default Code.gs.

3. Create All the Files (~26)

For each: + next to "Files" → Script or HTML → name it exactly → paste → save. Script (4): Code, DashboardModule, PTOModule, ReportsModule. Core HTML (6): Index, MainApp, MainAppJS, Styles, JavaScript, View_Help. View files (13): View_Dashboard, View_OTUpload, View_OTHistory, View_OTEmployee, View_OTTrends, View_UniformOrders, View_UniformDeductions, View_UniformCatalog, View_PTORecords, View_PayrollProcessing, View_SettingsPage, View_SystemHealth, View_PayrollCalendar, View_YearEndWizard. Forms (2): EmployeeUniformRequest, EmployeePTORequest.

4. Initialize the System

In Code.gs, pick manualInit in the function dropdown → Run. Authorize (Advanced → Go to Payroll System → Allow). The log lists the sheets it built. Safe to re-run.

5. Deploy the App

Deploy → New deployment → gear → Web app. Execute as Me, access Anyone (or your domain). Deploy, copy the URL, and bookmark it. First load is slow (10–20 sec). After any change, redeploy via Manage deployments → Edit → New version.

6. Configure Settings & Passcode

In the app's Settings: average hourly wage, OT multiplier (1.5), your locations, and admin emails; click Save All Settings. Passcodes are generated automatically the first time they're needed and emailed to your admins — change them any time in the Sheet's Payroll_Settings tab (manager_receiving_passcode and admin_access_passcode rows).

7. Add Employees

Easiest: Overtime → Upload Data with your CFA Home CSV — employees are created automatically. Or add them manually in the Employees tab.

8. Set Up the Employee Forms (QR Codes)

Add the page onto your Web app URL: YOUR-APP-URL?view=uniform-request and YOUR-APP-URL?view=pto-request. Make QR codes for each and post them by the time clock.

Use ?view=, not ?page=. The forms route off the view parameter. If a form link opens the dashboard instead, that's why.
Troubleshooting: "Script function not found: doGet" → re-copy Code.gs fully and redeploy. Blank screen → the Index file is missing/incomplete or a name's capitalization is off. "Authorization required" repeats → run any function once and Allow. Changes not showing → redeploy a new version. Form opens dashboard → use ?view=. Not sure → open System Health in the app.
