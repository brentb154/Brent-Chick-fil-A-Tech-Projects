Catering Quote Generator

Create, manage, email, and print professional catering quotes with a searchable menu picker, sequential quote IDs, and PDF email integration.

Google Apps Script Google Sheets Web App
This is a Google Apps Script application. It runs inside Google Sheets and cannot be opened directly in a browser. The full step-by-step setup guide walks you through deploying it to your Google account in about 25 minutes.
🧾 Jump to the Full Setup Guide ↓ 📖 Go deeper: what it is & how it works →

Features

  • Two restaurants, one tool — a Restaurant toggle on the quote form switches menu, cheat sheet, and pickup address per store (each store gets its own store-labeled tabs: CH Menu/DBU Menu and CH Off-Menu/DBU Off-Menu); tax rate is shared
  • Searchable, categorized menu item picker with live price display
  • Separate pickup and delivery pricing per item
  • Sequential quote IDs (Q-2026-0001, Q-2026-0002, …) that never repeat
  • Save & Email — generates a PDF and emails it in one click
  • Quote history with search, event-date filter, Edit & Reuse, and one-click Reorder for repeat customers
  • Customer memory — known organizations and their contacts autofill from past quotes
  • Revision history — every edit keeps the old version, viewable and restorable from the quote popup
  • Printable/emailable daily delivery runsheet straight from the calendar
  • Off-menu Cheat Sheet tab — team-editable add-on pricing with automatic delivery markup
  • Tax-exempt form tracking — on-file status per quote, one-click guest request email, Drive folder, year-end review reminder
  • Quote IDs carry the contact's initials (Q-2026-0042-KD); PDFs show quoted-on + valid-through dates
  • Optional automated day-before confirmation emails + daily missing-PO alert digest
  • "Valid through" expiration date on every quote PDF (Settings-tunable)
  • Calendar tab (Month / Week / Day) showing every order by event date, color-coded 🔴 needs PO / 🟢 PO handled
  • Optional customer phone; customer email and phone print on the PDF when present
  • Google Maps directions link on delivery quotes — PDF, quote popup, and calendar event
  • "Busy delivery window" warning before booking a 4th delivery within an hour
  • Quarterly price check — locks the tool every 90 days until 3 menu prices are verified against the POS
  • Fully customizable email subject and body templates
  • Optional BCC on every sent quote
  • Auto-archive moves quotes older than 120 days to a hidden sheet (nothing deleted)
  • Logo upload and per-location store details in Settings

Full Step-by-Step Setup Guide

About 25 minutes, no coding needed. You copy three files, run one setup function, add your menu, and deploy.

⚠️ Two rules that keep it working: don't create or rename tabs by hand (the setup function builds them), and make sure the HTML file holding App.html is named exactly Index. Those two mistakes cause almost every "it won't load."

1. Create the Google Sheet

At sheets.google.com, click Blank spreadsheet and name it "CFA Catering Quotes". Leave the tabs alone — Step 4 builds them.

2. Open Apps Script

Click Extensions → Apps Script. The script is bound to this Sheet, so you never paste a Sheet ID.

3. Paste the Three Files

Names must match exactly. Replace the starter Code.gs with the project's Code.gs; add an HTML file named Index holding App.html; add an HTML file named TaxForm holding TaxForm.html (the public guest upload page). (Upgrading an older install that still has Pipeline / PipelineView files? Delete both — the Pipeline feature was retired.) Save.

Most common mistake: the HTML files must be named exactly Index and TaxForm — doGet loads them by those names. Wrong name = blank page.

4. Run the Setup Function

Pick initializeSheet in the function dropdown → Run. Authorize when asked (Advanced → Go to (project) → Allow) — it needs Gmail, Drive, and Calendar to email PDFs and create events. This builds the visible tabs Settings, CH Menu + DBU Menu, Quotes, Quote_Sequence, CH Off-Menu + DBU Off-Menu, and Tax_Exempt_Registry, plus hidden log tabs (revisions, sent emails, tax uploads). Upgrading? It renames any old Menu/Off_Menu tabs to the CH-labeled ones in place, keeping your prices. It needs Gmail, Drive, and Calendar access. Safe to re-run.

5. Deploy as a Web App

Deploy → New deployment → gear → Web app. Execute as Me, access Anyone (this is required so external catering guests can reach the tax-form upload page). Deploy and copy the URL. Your team uses …/exec?view=app (bookmark it); guests get the ?view=taxform link automatically in the request email; the bare URL just shows a harmless landing page. First load is slow (10–20 sec). After any code change, redeploy via Manage deployments → Edit → New version.

6. Add Your Menu

The Menu tab has four columns: Category | Item Name | Pickup Price | Delivery Price. Same category groups items together in the picker, and N/A in Delivery Price hides an item from delivery quotes. You can type items in by hand, but the fastest way is to pull your real prices straight from CFA Home:

Pull your menu straight from CFA Home (recommended):

1. On CFA Home → Backoffice, search your restaurant by number. On its page, under Menu Price Group, click your price group (e.g. Price Group 32).

2. On the price list, click « Select a different price group.

3. On the Current Price Lists page, click your location link — e.g. "Location 03177 (Price Group 32)".

4. Click Export this to an Excel spreadsheet, then open the file, copy the menu rows, and paste them into the Menu tab of your Google Sheet.

The columns line up: Category → Category, Menu Item Name → Item Name, Inside Sales Price → Pickup Price, Outside Sales Price → Delivery Price. So paste them in that order under the existing headers. Any item with N/A for Outside Sales Price simply won't show on delivery quotes — which is exactly what you want.
Two restaurants? Do the same export for Dallas Baptist University and paste it into the DBU Menu tab (DBU's off-menu add-ons go on DBU Off-Menu). Cockrell Hill's live on CH Menu / CH Off-Menu. In the app, the Restaurant toggle above Order Type picks which store you're quoting, and the Menu Catalog / Cheat Sheet editors each have a Cockrell Hill / DBU switch. Set the store the form opens on under Settings → Default store for new quotes.

7. Fill in Settings

In the Settings tab, enter store name, address, phone, contact, tax rate, logo, email template, and (optionally) the follow-up reminder config. It's also where you tune operational settings like Archive After Days (default 120), Calendar Lead Time (Minutes) (default 30, how far before the order the calendar event starts), Price Check Interval (Days) (default 90, how often the tool locks for a price verification), Price Check Categories (which menu categories the spot-check draws from), Quote Valid For (Days) (the "valid through" date on PDFs), the delivery warning count/window, and the Daily Automation section — day-before confirmation emails and the missing-PO alert digest, each with its own on/off switch and a one-click "Enable (3pm daily)" button. The app reads all of these at runtime — no code edits.

8. Send a Test Quote

Open the Web app URL. First open shows a one-time price check — type the current POS prices for the 3 menu items it picks and it unlocks (it repeats quarterly to keep quote prices honest). Then build a quote and hit Save & Email to yourself. A PDF in your inbox and the order showing on the Calendar tab means you're live.

Optional Extras

Nightly archive of old quotes (moved to a hidden archive sheet, never deleted): add a time-driven trigger for cleanOldQuotes daily at midnight. How old before they archive is set in the Settings tab under Archive After Days (defaults to 120). Calendar events and follow-up reminder emails are separate feature deployments — see SETUP.md.

Troubleshooting: Blank page / "function not found" → the HTML file holding App.html isn't named Index. Tab missing → re-run initializeSheet. Menu items not showing → check the 4 columns, use N/A not blank. No email/PDF → re-authorize via initializeSheet. Changes not showing → redeploy a new version.
