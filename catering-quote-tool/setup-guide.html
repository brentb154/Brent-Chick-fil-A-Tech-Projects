Catering Quote Generator — Setup Guide

From zero to sending professional catering quotes. No coding experience needed — you copy four files, run one setup function, add your menu, and deploy. Follow the steps in order and you'll be done in about 25 minutes.

⏱ About 25 minutes

This tool lives inside one Google Sheet — there are no IDs or passwords to paste anywhere. You build quotes from your own menu, it numbers each one, makes a clean PDF, emails it in one click, and tracks every quote on a pipeline board from "sent" to "won."

How the pieces fit together (worth a quick read before you start):

  • The Sheet is both your database and your settings — menu, store info, and every saved quote live in its tabs.
  • The web app is the screen you and your team actually use to build and send quotes.
  • The pipeline board automatically tracks each emailed quote so nothing falls through the cracks.
⚠️ The two rules that keep it working: don't create or rename tabs by hand (the setup function builds them with the exact names the code expects), and make sure the file holding App.html is named exactly Index. Those two mistakes are behind almost every "it won't load" message.

Before You Start

  • A Google account that can create Google Sheets — your CFA Workspace account is ideal.
  • The four project files: Code.gs, App.html, Pipeline.gs, Pipelineview.html.
  • About 25 minutes, on a computer.
1

Create the Google Sheet

  1. Go to sheets.google.com and click Blank spreadsheet.
  2. Rename it "CFA Catering Quotes" (click the title, top-left).
  3. Leave the tabs alone — Step 4 builds the ones the tool needs.

This single spreadsheet holds everything: your menu, your store settings, and every quote you ever save. Don't delete it once you're live.

2

Open the Apps Script Editor

  1. In your Sheet, click Extensions → Apps Script. A code editor opens in a new tab.
  2. You'll see a starter Code.gs with an empty myFunction.

Because you opened the editor from inside the Sheet, the script is bound to it. That's why you never paste a Sheet ID — the code already knows which spreadsheet it belongs to.

3

Paste the Four Files

You need exactly four files in the editor. The app loads files by name, so match each one exactly, capitalization included:

Code (already there)ScriptCode.gs
IndexHTMLApp.html
PipelineScriptPipeline.gs
PipelineViewHTMLPipelineview.html

For new files, click the + next to "Files" and pick Script or HTML. Replace the starter Code.gs contents with the project's Code.gs (don't make a second one). Press Ctrl/Cmd+S to save.

The single most common mistake: the HTML file that holds App.html must be named exactly Index. The app's doGet loads the page by that name — if it's called App or index (lowercase), you'll get a blank page or a "script function not found" error with no other clue.
4

Run the Setup Function

  1. At the top of the editor, open the function dropdown and choose initializeSheet.
  2. Click ▶ Run.
  3. Google asks for permission the first time: Review permissions → pick your account → Advanced → Go to (project) → Allow.
  4. Check the Execution log at the bottom — no errors means it worked.

This builds your five tabs — Settings, Menu, Quotes, Quote_Sequence, and Pipeline — and seeds the starter structure.

Why it asks for so much access: the tool needs Gmail (to email quotes), Drive (to build PDFs), and Calendar (for the optional event feature). It's your own script, so granting it is safe. Safe to re-run too — initializeSheet re-creates any missing tab without touching data you've entered.
5

Deploy as a Web App

This turns the script into a real web page with a link your team opens.

  1. Click Deploy → New deployment (top-right).
  2. Click the gear ⚙ next to "Select type" and choose Web app.
  3. Set Execute as: Me and Who has access: Anyone within [your domain].
  4. Click Deploy, authorize if asked, and copy the Web app URL — that link is the tool. Bookmark it.

First load is slow (10–20 seconds) while Apps Script wakes up. That's normal.

Updating later: any time you change the code you must redeploy, or your users keep seeing the old version. Go to Deploy → Manage deployments → ✎ Edit → Version: New version → Deploy. The URL stays the same every time.
6

Add Your Menu

Open the Menu tab in the Sheet. It has four columns: Category | Item Name | Pickup Price | Delivery Price.

  • Categories are freeform — type anything (Trays, Box Meals, Sides, Drinks). Items sharing a category automatically group together in the picker.
  • Put N/A in the Delivery Price column for anything that can't be delivered — those items hide themselves automatically when someone builds a delivery quote.
Category   | Item Name                  | Pickup | Delivery

Drinks     | Gallon Iced Tea           | $11.00 | N/A
7

Fill in Settings

Open the Settings tab and enter your store name, address, phone, contact name, tax rate, logo, and email template. The app reads these at runtime, so editing the Sheet is all it takes — you never touch code to change a setting. This is also where you configure the follow-up reminder emails (see Optional Extras).

8

Send a Test Quote

  1. Open your Web app URL from Step 5.
  2. Build a quote, pick a couple of menu items, and click Save & Email to send it to yourself.
  3. Confirm the PDF arrives, and that the quote shows up under History and on the Pipeline board.
✅ A PDF in your inbox and a new card on the pipeline means you're live.

What Each Tab Does

TabWhat it holds
SettingsStore details, tax rate, logo, email templates, reminder config. The tab you'll edit most.
MenuYour items and pricing (4 columns). Drives the searchable picker.
QuotesEvery saved quote, one row each. Prices are frozen here at the moment you create the quote.
Quote_SequenceTracks the next quote number so IDs never repeat. Leave it alone.
PipelineThe sales board — every emailed quote, tracked by stage.
Don't rename any of these. The code looks each one up by its exact name; rename one and that feature quietly stops working.

How Quotes & the Pipeline Work

  • Sequential IDs — every quote gets a number like Q-2026-0001 that never repeats, tracked in Quote_Sequence.
  • Prices freeze at creation — once a quote is saved, changing your Menu prices later never alters quotes you already sent.
  • Edit & Reuse — reworking an old quote creates a new one; the original is never modified.
  • The pipeline — when you email a quote it lands on the board, and you move it through Sent → Confirmed → Won (or Cancelled). The cards show counts and totals per stage so you can see what's still open.
  • Auto-cleanup — quotes older than 30 days are removed automatically once you set up the optional nightly trigger below.

Optional Extras

Nightly cleanup — to auto-remove quotes older than 30 days, add a time-driven trigger (clock icon in the editor) for cleanOldQuotes set to run daily around midnight.

Calendar events & follow-up reminders — auto-creating a Google Calendar event when a quote is saved, and sending follow-up reminder emails, are separate feature deployments with their own steps and authorization. They're written up in SETUP.md.

Where Your Team Opens It

It's a web app, so it's just a link — no install. Open the Web app URL from Step 5 and:

  • Bookmark it on the catering coordinator's computer.
  • Share the link with whoever builds quotes — access is already limited to your domain from the deploy settings.

Troubleshooting

Web app shows a blank page or "script function not found" — the HTML file holding App.html isn't named exactly Index (Step 3). Rename it and redeploy a new version.

"getSheetByName(...) is null" or a tab is missing — run initializeSheet again (Step 4). It re-creates any missing tab without touching existing data.

Menu items don't show in the picker — check the Menu tab has all four columns (Category, Item Name, Pickup Price, Delivery Price). Use N/A (not blank) for non-deliverable items' delivery price.

Quote saves but no email or PDF arrives — re-authorize: run initializeSheet once from the editor and click Allow (the script needs Gmail + Drive access).

Calendar events or reminders aren't working — those are the optional features in SETUP.md; see its troubleshooting section.

Changes to the code don't show up — you saved but didn't redeploy. Deploy → Manage deployments → Edit → New version → Deploy.
