Catering Quote Generator — Setup Guide

From zero to sending professional catering quotes. No coding experience needed — you copy a few files, run one setup function, and add your menu. Follow the steps in order.

⏱ About 25 minutes
This tool lives inside one Google Sheet — there are no IDs or passwords to paste. You build quotes from your own menu, it numbers them, makes a clean PDF, emails it, and tracks every quote on a pipeline board.
⚠️ The one rule that keeps it working: don't create or rename tabs by hand. The setup function builds Settings, Menu, Quotes, Quote_Sequence, and Pipeline with the exact names the code expects. Rename one and that feature quietly stops working.
1

Create the Google Sheet

  1. Go to sheets.google.com and click Blank spreadsheet.
  2. Rename it "CFA Catering Quotes" (click the title, top-left).
  3. Leave the tabs alone — Step 4 builds them for you.
2

Open the Apps Script Editor

  1. In your Sheet, click Extensions → Apps Script. A code editor opens in a new tab.
  2. The script is now bound to your Sheet — that's why you never paste a Sheet ID.
3

Paste the Four Files

You need exactly four files in the editor. Match each name exactly, capitalization included:

Create this fileTypePaste the contents of
Code (already there)ScriptCode.gs
IndexHTMLApp.html
PipelineScriptPipeline.gs
PipelineViewHTMLPipelineview.html

For new files use the + next to "Files" → Script or HTML. Replace the starter Code.gs contents with the project's Code.gs. Click 💾 Save.

Watch this one: the HTML file that holds App.html must be named exactly Index — doGet loads it by that name. If the page comes up blank later, this is almost always why.
4

Run the Setup Function

  1. At the top of the editor, open the function dropdown and choose initializeSheet.
  2. Click ▶ Run.
  3. Google asks for permission the first time: Review permissions → pick your account → Advanced → Go to (project) → Allow. (It needs Gmail, Drive, and Calendar so it can email PDFs and create events.)
  4. Check the Execution log — no errors means you're good.

Your Sheet now has five tabs: Settings, Menu, Quotes, Quote_Sequence, and Pipeline.

Safe to re-run. initializeSheet re-creates any missing tab without touching data you've already entered.
5

Deploy as a Web App

  1. Click Deploy → New deployment (top-right).
  2. Gear ⚙ next to "Select type" → Web app.
  3. Set Execute as: Me and Who has access: Anyone within [your domain].
  4. Click Deploy, authorize if asked, and copy the Web app URL — that's the link your team opens.
Updating later: after any code change go to Deploy → Manage deployments → ✎ Edit → Version: New version → Deploy. The URL stays the same. Saving code alone does nothing until you redeploy a new version.
6

Add Your Menu

Open the Menu tab. It has four columns: Category | Item Name | Pickup Price | Delivery Price.

  • Categories are freeform — type anything (Trays, Box Meals, Sides, Drinks). Items with the same category group together in the picker.
  • Put N/A in Delivery Price for anything that can't be delivered — it'll hide automatically when someone picks Delivery.
7

Fill in Settings

Open the Settings tab and fill in your store name, address, phone, contact name, tax rate, logo, and email template. The app reads these at runtime — you never edit code to change them.

8

Send a Test Quote

  1. Open your Web app URL from Step 5.
  2. Build a quote, pick a couple menu items, and hit Save & Email to yourself.
  3. Confirm the PDF arrives and the quote shows up under History and on the Pipeline board.
✅ PDF in your inbox + quote on the pipeline = you're live.

Optional Extras

Nightly cleanup — to auto-delete quotes older than 30 days, add a time-driven trigger (clock icon in the editor) for cleanOldQuotes set to run daily around midnight.

Calendar events & follow-up reminders — these are separate feature deployments with their own steps and authorization. See SETUP.md for those.

Troubleshooting

Web app shows a blank page or "script function not found" — the HTML file holding App.html isn't named exactly Index (Step 3). Rename it and redeploy.

"getSheetByName(...) is null" / a tab is missing — run initializeSheet again (Step 4). It re-creates any missing tab without touching existing data.

Menu items don't show in the picker — check the Menu tab has all four columns. Use N/A (not blank) for non-deliverable items' delivery price.

Quote saves but no email or PDF — re-authorize: run initializeSheet once from the editor and click Allow (it needs Gmail + Drive access).

Calendar events or reminders not working — see SETUP.md — those are the feature-specific deployment steps and their troubleshooting.

Changes to the code don't show up — you edited but didn't redeploy. Deploy → Manage deployments → Edit → New version → Deploy.
